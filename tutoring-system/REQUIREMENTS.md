# REQUIREMENTS.md

EPQ 学术辅导进度管理系统 · 需求文档  
版本：v0.1 · 日期：2026-04-09

---

## 背景

系统供一名 EPQ 学术辅导教师（TA）个人使用，用于管理最多 30 名学生的辅导进度。每位学生的研究课题各不相同。课程分三类：SA 会议（课时有限，需精确计数）、TA 会议、理论课。偶尔需要向家长或市场部导出格式化摘要。

---

## 一、核心功能

### 1. 学生管理

- 每位学生拥有独立档案，包含：
  - 基本信息：姓名、研究课题
  - SA 课时：额度上限、已用课时、剩余课时
  - 自定义标签（可增删，全局统一管理标签库）
  - 时间窗口备注（如"六月前专注备考，暂停辅导"）——支持显示提醒
  - 私人备注（仅自己可见，不进入任何导出内容）

### 2. 课程记录

- 三类课程（SA 会议 / TA 会议 / 理论课）共用同一记录表单，字段包含：
  - 课程类型、日期、时长
  - 课程内容摘要
  - 布置的作业 / 下次目标
  - 会议文字纪要（手动粘贴文字，预留将来文件上传 + API 接入的扩展点）
- SA 会议记录自动累计已用课时，剩余课时实时更新

### 3. Dashboard（总览卡片）

- 以卡片形式展示所有学生，每张卡片显示：
  - 姓名、研究课题
  - 上次上课日期（区分课程类型）
  - 下次上课时间（区分 SA / TA / 理论课）
  - SA 剩余课时
  - 标签
  - 简短备注（单行）
- 支持按标签筛选、按"上次见面时间"排序，帮助识别需要跟进的学生

### 4. EPQ 里程碑追踪

- 里程碑节点固定如下（顺序即为 EPQ 标准流程），带"如有"标注的节点可按学生实际情况勾选跳过：

  ```
  问卷（如有）→ Intro → 文综 → 方法论 → 结果 → 讨论 → 反思 → 结语 →
  文献 → 摘要 → 表1 → 表2 → 表4 → 表5 → 表6 → 表7 → 表11 → 答辩 → 提交
  ```

- 每位学生独立设置各里程碑的完成状态（未开始 / 进行中 / 已完成）
- 进度条形式呈现，进入导出内容
- "问卷"节点标记为可选（不适用时不计入进度百分比）

### 5. 导出 / 分享

- 生成格式化文字摘要（含 Emoji），适合发送给家长或市场部
- 导出内容包含：课题、里程碑进度、课程记录摘要、SA 课时使用情况
- **私人备注字段不进入任何导出内容**
- 导出为可复制文字（v1），将来可扩展为 PDF 或分享链接

## 二、非功能需求

| 维度 | 要求 |
|------|------|
| 数据存储 | 数据以 JSON 文件存储在私有 GitHub 仓库，不使用任何第三方数据库或 SaaS 存储 |
| 访问控制 | 通过 GitHub Personal Access Token（PAT）鉴权，Token 存于本地浏览器，相当于系统密码 |
| 跨设备同步 | 支持 macOS + Windows，打开网页输入 PAT 即可访问最新数据 |
| 托管方式 | 初期部署于 GitHub Pages（静态托管），后期可迁移至自购公网服务器 |
| 目标规模 | 以 30 名学生为设计上限，当前约 7 名 |
| 可延展性 | 数据层、认证层、会议纪要导入方式均须设计为可替换，不与 GitHub API 强耦合 |
| 移动端 | 暂不需要，但 UI 布局需保留响应式基础，方便日后扩展 |

---

## 三、仓库结构

| 仓库 | 用途 | 可见性 |
|------|------|--------|
| `epq-tutor-dashboard`（代码仓库） | 前端源码 + GitHub Pages 部署 | **公开**（可向同事展示） |
| `epq-tutor-data`（数据仓库） | 所有学生 JSON 数据文件 | **私有** |

前端通过 GitHub Personal Access Token（PAT）调用 GitHub API 读写数据仓库。PAT 存于浏览器 localStorage，相当于系统登录密码。

---

## 四、明确不做（v1 范围外）

- 学生端门户（学生自己登录查看）
- 多用户 / 角色权限系统
- 自动接入 Zoom API 抓取会议纪要（预留扩展点，手动粘贴为 v1）
- 文件上传（图片、Word、PDF 等），v1 只支持纯文本粘贴
- 自动发送提醒（邮件 / 短信 / 微信）
- 财务结算 / 发票管理（SA 课时记录仅供工资核对参考，不做计算）
- 日历集成（Google Calendar / iCal）
- 实时协作编辑
